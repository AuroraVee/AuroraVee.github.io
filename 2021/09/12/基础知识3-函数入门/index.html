    函数入门

    1.声明函数

    在 JS 里，声明函数只有 3 种方法：

    //第1种 函数声明
 function foo(){...} 
 
//第2种 函数表达式
 var foo = function(){...} 
 
//第3种 箭头函数
 let f=(variable)=>{}

    除此之外，类似于 var foo = function bar(){...} 这样的统一按第 2 种情况处理，即在函数外部无法通过 bar 访问到函数，因为这已经变成了一个表达式。

    2.函数的arguments 对象

    Arguments 对象只定义在函数体中，包括了函数的参数和其他属性。在函数体中，arguments 指代该函数的 Arguments 对象。

    属性：

    1. length属性

    arguments.length：返回的是函数的实参个数

    函数名.length：返回的是函数的形参个数

    1. callee属性

    该属性是一个指针，指向拥有这个arguments对象的函数。一般用于递归时调用函数本身

    function f(num){
    if(num<1){
        return 1;
    }else{
        return num*arguments.callee(num-1);
    }
}

    另外callee属性还有一个caller属性，这个属性保存着调用当前函数的函数的引用。

    function outer(){
    inner()
}
function inner(){
    console.log(arguments.callee.caller)
}
    1. arguments和对应参数的绑定

    有实参对应的形参和arguments是共享参数的，即其中一个改变则另一个也会随之改变；而形参没有接收到参数时，两者不共享。除此之外，以上是在非严格模式下，如果是在严格模式下，实参和 arguments 是不会共享的。

    function foo(name, age, sex, hobbit) {

    console.log(name, arguments[0]); // name name

    // 改变形参
    name = 'new name';

    console.log(name, arguments[0]); // new name new name

    // 改变arguments
    arguments[1] = 'new age';

    console.log(age, arguments[1]); // new age new age

    // 测试未传入的是否会绑定
    console.log(sex); // undefined

    sex = 'new sex';

    console.log(sex, arguments[2]); // new sex undefined

    arguments[3] = 'new hobbit';

    console.log(hobbit, arguments[3]); // undefined new hobbit

}

foo('name', 'age')

    3.原型与原型链

    参考资料:https://github.com/mqyqingfeng/Blog/issues/2

    1. js中每一个对象或函数对象都有__proto__属性，但是只有函数对象才有prototype属性。
    2. Object.prototype只是一个普通对象，它是js原型链的最顶端
    Object.prototype.__proto__=== null;//true

Object.prototype.prototype === undefied;//true
    1. 实例对象的__proto__属性指向它构造函数的prototype
    function f(){}
var obj=new f();
obj.__proto__==f.prototype  //true 即实例对象的原型对象为构造函数的prototype，即它是一个对象
    1. 内置的Object也是一个函数对象，它是由Function创建的。
    Object.__proto__ === Function.prototype;
    1. Function.prototype是个特例，它是函数对象，但是没有prototype属性。其他所有函数都有prototype属性。
    Function.prototype.prototype === undefined; //true
    1. 内置的Function也是一个函数对象，它是通过自己来创建自己的
    Function.__proto__=== Function.prototype; //true
    1. 函数也是对象，因为Function.prototype.__proto__指向Object.prototype
    typeof Function.prototype.[[proto]] === "object";//true

Function.prototype.[[proto]]=== Object.prototype;//true
    1. 普通对象的原型对象
    var obj={};  //obj.__proto__==Object.prototype  true
var x={w:1};
Object.setPrototypeOf(obj,x);  //相当于obj.__proto=x;

    4.作用域链和词法作用域

    优先选择当前所在的作用域，如果当前作用域不存在需要的变量就往上一级查找

    var x = 1;//1级：全局作用域
function foo(x, y = function() { x = 2; }) {//2级：函数参数形成一个单独的作用域
  var x = 3;//3级：函数体内是一个单独的作用域
  y();
  console.log(x);//函数体内的x
}

foo() // 3
x // 1

    JavaScript采用的是词法作用域（静态作用域），函数的作用域基于函数创建的位置

    5.函数调用执行的内部情况

    参考资料：https://github.com/mqyqingfeng/Blog/

    5.1变量提升与函数提升

    1. var变量声明会被提升在作用域顶端
    2. 变量定义，声明部分会被提升，赋值部分不会被提升！
    3. 函数表达式，只提升变量名，不提升函数值
    4. 函数声明，函数名和函数体一起提升，优先级比变量声明要高，名字相同的变量声明会被忽略！

    5.2 执行上下文栈

    https://github.com/mqyqingfeng/Blog/issues/5

    JavaScript并不是一行行顺序执行的，涉及到变量提升和函数提升。另外，当JavaScript运到可执行代码时（全局代码、函数代码、eval代码），JavaScript引擎会创建执行上下文栈，每个函数在要执行的时候都会创建对应的执行上下文，并压入栈中，当执行结束后（即return的时候）就从栈里弹出。

    对于每个执行上下文，都有三个重要属性：

    • 变量对象(Variable object，VO)
    • 作用域链(Scope chain)
    • this

    5.2.1 变量对象

    与执行上下文相关的数据作用域，存储了在上下文中定义的变量和函数声明。全局上下文中的变量对象就是全局对象。函数上下文中的变量对象只有在进入它的执行上下文时才被激活（叫作活动对象），这个时候活动对象的各种属性才能被访问。

    1. 全局上下文的变量对象初始化是全局对象
    2. 函数上下文的变量对象初始化只包括 Arguments 对象
    3. 在进入执行上下文时会给变量对象添加形参、函数声明、变量声明等初始的属性值
    4. 在代码执行阶段，会再次修改变量对象的属性值

    进入执行上下文：当进入执行上下文时，这时候还没有执行代码

    变量对象会包括：

    1. 函数的所有形参 (如果是函数上下文)
      • 由名称和对应值组成的一个变量对象的属性被创建
      • 没有实参，属性值设为 undefined
    2. 函数声明
      • 由名称和对应值（函数对象(function-object)）组成一个变量对象的属性被创建
      • 如果变量对象已经存在相同名称的属性，则完全替换这个属性
    3. 变量声明
      • 由名称和对应值（undefined）组成一个变量对象的属性被创建；
      • 如果变量名称跟已经声明的形式参数或函数相同，则变量声明不会干扰已经存在的这类属性
      • 即在进入执行上下文时，首先会处理函数声明，其次会处理变量声明

    注：函数中的变量如果没有通过 var、let、const 关键字声明，不会被存放在 AO 中。只能在运行阶段作为全局对象的属性使用。

    function foo(a) {
  var b = 2;
  function c() {}
  var d = function() {};

  b = 3;

}

foo(1);

    //进入执行上下文后的活动对象为
AO = {
    arguments: {
        0: 1,
        length: 1
    },
    a: 1,
    b: undefined,//此时b只声明
    c: reference to function c(){},
        //普通函数声明时就有初始值
    d: undefined //函数表达式只声明没有初始值
}

    5.2.2 作用域链

    查找变量时，先从当前上下文的变量对象开始查找，找不到再找父级上下文的变量对象。这样由多个执行上下文形成的变量对象构成的链表叫做作用域链。

    5.3 函数的生命周期

    var scope = "global scope";
function checkscope(){
    var scope2 = 'local scope';
    return scope2;
}
checkscope();
    1. 定义时：确定函数创建时所在的作用域。函数对象的[[scope]]属性将会保存父级上下文的变量对象VO
    checkscope.[[scope]]=[globalContext.VO]
    1. 执行时：创建执行上下文，并压入执行上下文栈中
    ECStack = [
    checkscopeContext,
    globalContext
];

    1. 函数并不立刻开始执行，开始做准备工作：

    2. 复制函数的[[scope]]属性创建作用域链（scope）

    checkscopeContext = {
    Scope: checkscope.[[scope]],
}
    1. 用 arguments 创建活动对象，随后初始化活动对象，加入形参、函数声明、变量声明
    checkscopeContext = {
    AO: {
        arguments: {
            length: 0
        },
        scope2: undefined
    }，
    Scope: checkscope.[[scope]],
}
    1. 将活动对象压入 checkscope 作用域链顶端
    checkscopeContext = {
    AO: {
        arguments: {
            length: 0
        },
        scope2: undefined
    },
    Scope: [AO, [[Scope]]]
}
    1. 开始执行函数：随着函数的执行，修改 AO 的属性值
    checkscopeContext = {
    AO: {
        arguments: {
            length: 0
        },
        scope2: 'local scope'
    },
    Scope: [AO, [[Scope]]]
}
    1. 执行结束：函数上下文从执行上下文栈中弹出
    ECStack = [
    globalContext
];

    6. 闭包

    ECMAScript中，闭包指的是：

    1. 从理论角度：所有的函数。因为它们都在创建的时候就将上层上下文的数据保存起来了。哪怕是简单的全局变量也是如此，因为函数中访问全局变量就相当于是在访问自由变量，这个时候使用最外层的作用域。
    2. 从实践角度：以下函数才算是闭包：
      • 即使创建它的上下文已经销毁，它仍然存在（比如，内部函数从父函数中返回）
      • 在代码中引用了自由变量

    自由变量：在函数中使用的，但既不是函数参数也不是函数的局部变量的变量。

    实现原理：闭包在执行的时候保存一个作用域链，可以利用匿名函数的上下文暂存变量的值

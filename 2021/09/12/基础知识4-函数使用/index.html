    函数使用

    1.this使用

    1. 当在全局进行函数调用的时候指向window，严格模式下函数调用，this=undefined

    2. 当方法调用的时候指向调用对象

    3. 当用apply和call上下文调用的时候指向传入的第一个参数

    4. 构造函数调用指向实例对象

    5. 箭头函数不会创建自己的this，它只会从自己的作用域链的上一层继承this

    6. 事件绑定：addEventListener指向当前绑定的元素，attachEvent指向全局window

    2.apply、call、bind

    用途:手动改变this的指向

    区别：

    1. apply和call会使当前函数立即执行，bind会返回一个函数，后续需要时再调用
    2. call中要传多个任意参数，apply只可以直接传数组或者类数组
    3. bind是为函数绑定一个this上下文

    如果上下文的值为null，则使用全局对象代替

    规则:

    1. fn.apply(上下文环境，执行所需数组)
    2. fn.call(上下文环境，执行所需单个参数)
    3. fn.bind(上下文环境)

    模拟实现：

    3.函数柯里化

    柯里化是一种将使用**多个参数的一个函数转换成一系列使用一个参数的函数的技术，这些函数每次调用的时候都只传一个参数。将一个 n 元函数转换成 n 个一元函数。意义：可以复用一些参数**，比如每次调用的时候，前3个参数始终不变，只有最后一个参数不一样

    应用场景：函数式编程

    //简单版 实参的顺序和形参的顺序保持一致
      function sub(fn) {//自动解构赋值 combined[0]
        let arg = [].slice.call(arguments, 1);
        return function () {
          fn.apply(this, arg.concat([].slice.call(arguments)));
        };
      }

      function curry(fn, length) {
        length = length || fn.length;//未赋值的参数的个数
        return function () {
          if (arguments.length < length) {
            let combined = [fn].concat(Array.from(arguments));//[fn,"a","b"]
            let temp = sub.apply(this, combined);
            return curry(temp, length - arguments.length);
          } else {
            fn.apply(this, arguments);
          }
        };
      }

//验证
      var fn = curry(function (a, b, c) {
        console.log([a, b, c]);
      });

      fn("a", "b", "c"); // ["a", "b", "c"]
      fn("a", "b")("c"); // ["a", "b", "c"]
      fn("a")("b")("c"); // ["a", "b", "c"]
      fn("a")("b", "c"); // ["a", "b", "c"]

    4.偏函数

    局部应用则是固定一个函数的一个或者多个参数，也就是**将一个 n 元函数转换成一个 n - x 元函数**。

    // 第一版 不改变this指向
function partial(fn) {
    var args = [].slice.call(arguments, 1);
    return function() {
        var newArgs = args.concat([].slice.call(arguments));
        return fn.apply(this, newArgs);
    };
};

// 第二版
var _ = {};

function partial(fn) {
    var args = [].slice.call(arguments, 1);
    return function() {
        var position = 0, len = args.length;
        for(var i = 0; i < len; i++) {
            args[i] = args[i] === _ ? arguments[position++] : args[i]
            //把第一次传的参数里面为_的值替换成第二次参数里对应的值
        }
        while(position < arguments.length) args.push(arguments[position++]);//把第二次剩下的没传的参数都放在args
        return fn.apply(this, args);
    };
};

//验证
var subtract = function(a, b) { return b - a; };
subFrom20 = partial(subtract, _, 20);
subFrom20(5);

    5.惰性函数

    惰性函数就是解决每次都要进行判断的这个问题，解决原理很简单，重写函数

    当我们每次都需要进行条件判断，其实只需要判断一次，接下来的使用方式都不会发生改变的时候，想想是否可以考虑使用惰性函数。

    //情景1：需要写一个 foo 函数，这个函数返回首次调用时的 Date 对象
var foo = function() {
    var t = new Date();
    foo = function() {
        return t;
    };
    return foo();
};

//情景2
function addEvent (type, el, fn) {
    if (window.addEventListener) {
        addEvent = function (type, el, fn) {
            el.addEventListener(type, fn, false);
        }
    }
    else if(window.attachEvent){
        addEvent = function (type, el, fn) {
            el.attachEvent('on' + type, fn);
        }
    }
    
    addEvent(type, el, fn) //调用一次，给目标元素绑定好事件
}

    6.函数组合

    把多个函数组合成一个函数，在调用的时候可以直接传入最右边的一个函数需要的参数，不必多次显示调用

    //从右往左依次调用
function compose() {
    var args = arguments;
    var start = args.length - 1;
    return function() {
        var i = start;
        var result = args[start].apply(this, arguments);
        while (i--) result = args[i].call(this, result);
        return result;
    };
};

//验证
var greet = compose(hello, toUpperCase);
greet('kevin');

    具体应用：函数组合可以将多种基本运算函数自由的组合形成一个功能强大的函数，这个函数不直接处理数据即pointfree。然后在实际场景下调用这个最终的函数

    Pointfree 的本质就是使用一些通用的函数，组合出各种复杂运算。上层运算不要直接操作数据，而是通过底层函数去处理。即不使用所要处理的值，只合成运算过程。

    pointfree 模式能够帮助我们减少不必要的命名，让代码保持简洁和通用，更符合语义，更容易复用，测试也变得轻而易举

    // 需求：输入 'kevin daisy kelly'，返回 'K.D.K'

// 非 pointfree，因为提到了数据：name
var initials = function (name) {
    return name.split(' ').map(compose(toUpperCase, head)).join('. ');
};

// pointfree
// 先定义基本运算
var split = curry(function(separator, str) { return str.split(separator) })
var head = function(str) { return str.slice(0, 1) }
var toUpperCase = function(str) { return str.toUpperCase() }
var join = curry(function(separator, arr) { return arr.join(separator) })
var map = curry(function(fn, arr) { return arr.map(fn) })

var initials = compose(join('.'), map(compose(toUpperCase, head)), split(' '));

initials("kevin daisy kelly");

    7.函数记忆

    将上次的计算结果缓存起来，当下次调用时，如果遇到相同的参数，就直接返回缓存中的数据。比如递归的时候

    函数记忆只是一种编程技巧，本质上是**牺牲算法的空间复杂度以换取更优的时间复杂度，在客户端 JavaScript 中代码的执行时间复杂度往往成为瓶颈，因此在大多数场景下**，这种牺牲空间换取时间的做法以提升程序执行效率的做法是非常可取的

    let memorize=function(func){
    let cache={}
    return function(){
        let key=arguments.length+JSON.stringify(arguments)//用JSON.stringify可以解决传入的参数为对象时，转换成字符串成为[Object object]的情况
        if(key in cache){//不写if(!cache[key])，可以避免cache[key]===0的情况
            return cache[key]
        }else{
            cache[key]=func.apply(this,arguments)
            return cache[key]
        }
    }
}

//不使用函数记忆
let count = 0;
let fibonacci = function(n){
    count++;
    return n < 2? n : fibonacci(n-1) + fibonacci(n-2);
};
for (var i = 0; i <= 10; i++){
    fibonacci(i)
}
console.log(count) // 453

//使用函数记忆
let fibonacci=memorize(fibonacci)
count = 0;
for (var i = 0; i <= 10; i++){
    fibonacci(i)
}
console.log(count) // 11

    8.递归

    程序调用自身的编程技巧称为递归(recursion)。

    构成递归需具备边界条件、递归前进段和递归返回段。当边界条件不满足时，递归前进，当边界条件满足时，递归返回。

    普通递归

    当执行一个函数的时候，就会创建一个执行上下文，并且压入执行上下文栈，当函数执行完毕的时候，就会将函数的执行上下文从栈中弹出。

    当递归时，JavaScript 会不停的创建执行上下文压入执行上下文栈，对于内存而言，维护这么多的执行上下文开销很大，可以使用尾调用来优化。

    function fibonacci(n){
    return n < 2 ? n : fibonacci(n - 1) + fibonacci(n - 2);
}
console.log(fibonacci(5))

    尾调用

    尾调用，是指函数内部的最后一个动作是函数调用。该调用的返回值，直接返回给函数

    比如：

    // 尾调用
function f(x){
    return g(x);
}
//执行栈变化
ECStack = [];
ECStack.push(<f> functionContext);
ECStack.pop();
ECStack.push(<g> functionContext);
ECStack.pop();


// 非尾调用
function f(x){
    return g(x) + 1;
}
//执行栈变化
ECStack = [];
ECStack.push(<f> functionContext);
ECStack.push(<g> functionContext);
ECStack.pop();
ECStack.pop();

    尾递归

    函数调用自身，称为递归。如果尾调用自身，就称为尾递归

    function factorial(n, res) {
    if (n == 1) return res;
    return factorial(n - 1, n * res)
}

console.log(factorial(4, 1)) // 24

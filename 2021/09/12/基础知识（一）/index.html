    4.原生具有iterator的数据结构

    ES6规定，默认的Iterator接口部署在数据结构的Symbol.iterator属性，或者说，一个数据结构只要具有Symbol.iterator属性，就可以认为是“可遍历的”(iterable) 。其中满足条件的数据结构有：

    • Array

    • Map

    • Set

    • String

    • TypedArray：Int8Array、Uint8Array、Uint8ClampedArray、Int16Array、Uint16Array、Int32Array、Uint32Array、Float32Array、Float64Array

    • Generator 函数返回的对象

    • 函数的 arguments 对象

    • NodeList 对象

      5.类数组对象

      5.1 类数组对象

    1. DOM 操作返回的NodeList 集合
    2. 函数内部的arguments对象
    3. 具有length属性的对象，且以数字作为属性名，有效的属性范围[0,length-1]

    其中第三种本来是没有iterator接口，但是可以用Array.from或者扩展运算符转化为数组后，进行遍历

    5.2 转化为数组

    将类数组对象转为数组的方法有：

    Array.from(arr);//将类似数组的对象（array-like object）和可遍历（iterable）的对象转为数组
Array.prototype.slice.call(arrayLike);
Array.prototype.splice.call(arrayLike, 0);
Array.prototype.concat.apply([], arrayLike)
[...arr];//arr 具有iterator接口的对象

    //举例说明
obj={length:2,0:"w",2:"e"};
Array.from(obj);// ["w", undefined]
Array.from({ length: 3 });// [ undefined, undefined, undefined ]

//将传入的参数转成数组：
Array.of() //总是返回参数值组成的数组
0 in [, , ,] // false 表示求数组的0号位是否有值

    6.数据存储

    indexDB的出现 、web-storage的限制参照

    特性 Cookie storage localStorage sessionStorage globalstorage
    数据的生命期 一般由服务器生成，可设置失效时间。如果在浏览器端生成Cookie，默认是关闭浏览器后失效 除非被清除，否则永久保存（取代globalstorage） 仅在当前会话下有效，关闭页面或浏览器后被清除 如果用户未清除浏览器的缓存，数据将一直存储在磁盘上
    存放数据大小 4095B左右 提供最大的存储空间 一般为5MB 针对会话的小段数据存储
    与服务器端通信 每次都会携带在HTTP头中，如果使用cookie保存过多数据会带来性能问题 不需要将数据发回服务器 仅在客户端（即浏览器）中保存，不参与和服务器的通信 不需要将数据发回服务器 不需要将数据发回服务器
    易用性 需要程序员自己封装，源生的Cookie接口不友好 源生接口可以接受，亦可再次封装来对Object和Array有更好的支持 不能跨会话存储数据 跨适合客户端存储文档或者长期保存用户偏好设置

    7.JavaScript模块化

    • 浏览器端：
    1. AMD：异步模块规范，依赖前置，由require.js实现。允许指定回调函数
    2. CMD：通用模块规范，依赖就近，由sea.js实现
    • 服务器端：CommonJS，由NodeJs实现（node为JavaScript执行提供了运行环境，不需要借助浏览器即可运行，V8引擎将JavaScript转成了二进制）。同步的，只有加载完成，才能执行后面的操作

    浏览器不兼容CommonJS的根本原因，在于缺少四个Node.js环境的变量。

    • module
    • exports
    • require
    • global

    浏览器加载 CommonJS，需要进行格式转换，工具Browserify 、Tiny Browser Require

    8. iframe的局限

    1. 创建比一般的 DOM 元素慢了 1-2 个数量级
    2. 阻塞页面加载
    3. 唯一的连接池
    4. 不利于 SEO(搜索引擎优化)

    9.页面的性能指标

    • 白屏时间

    指浏览器发起请求到开始显示第一个页面元素的时间。现代浏览器不会等待CSS树（所有CSS文件下载和解析完成）和DOM树（整个body标签解析完成）构建完成才开始绘制，而是马上开始显示中间结果。所以经常在低网速的环境中，观察到页面由上至下缓慢显示完，或者先显示文本内容后再重绘成带有格式的页面内容。

    • 首屏时间

    首屏时间（FirstScreen Time），是指用户看到第一屏，即整个网页顶部大小为当前窗口的区域，显示完整的时间。常用的方法有，页面标签标记法、图像相似度比较法和首屏高度内图片加载法。

    • 可交互时间

    用户可以进行正常的点击、输入等操作，默认可以统计DOMContentLoaded事件发生的时间。

    • 整页时间

    整页时间（Page Load Time），页面所有资源都加载完成并呈现出来所花的时间，这个就是load事件发生的时间。

    10.动态语言与静态语言

    • 动态语言（弱类型语言）

    在运行时确定数据类型的语言，变量使用之前不需要类型声明。如PHP/ASP/RUBY/PYTHON/JavaScript

    • 静态语言（强类型语言）

    在编译时变量的类型即可确定的语言，要求在使用之前必须声明数据类型。如C++/Java/C#

    11.行为委托(面向委托的设计模式)

    11.1 对象关联委托实现

    和其他语言的类设计模式不一样

    //对象关联风格

Task = { 
 setID: function(ID) { this.id = ID; }, 
 outputID: function() { console.log( this.id ); } 
}; 

// XYZ 委托了Task对象 
XYZ = Object.create( Task ); //对象关联
//Object.setPrototypeOf( XYZ, Task ); 这种方式也可以

XYZ.prepareTask = function(ID,Label) { 
    //直接通过this调用委托对象的函数
 this.setID( ID ); 
 this.label = Label; 
}; 
XYZ.outputTaskDetails = function() { 
 this.outputID(); 
 console.log( this.label ); 
}; 
// ABC = Object.create( Task );
// ABC ... = ...

    XYZ对象通过行为委托[[prototype]]和Task对象关联起来。委托行为意味着某些对象在找不到属性或方法时会把这个请求委托给另一个对象。

    与类设计模式不一样的是，这种直接通过对象的方式委托，不再需要new（构造函数）的方式创建实例来调用，而且子类调用父类的时候只需要简单的this调用即可，不再需要通过显示的prototype、call、bind等复杂的写法。代码和逻辑都更加简单

    //简写的对象关联，可以不写function，但是这种方法在函数自我引用时不方便，还是应该采用具名函数表达式
Task = { 
 setID(ID) { this.id = ID; }, 
 outputID() { console.log( this.id ); } 
}; 

XYZ = Object.create( Task ); //对象关联

XYZ.prepareTask = function(ID,Label) { 
    //直接通过this调用委托对象的函数
 this.setID( ID ); 
 this.label = Label; 
}; 
XYZ.outputTaskDetails = function() { 
 this.outputID(); 
 console.log( this.label ); 
};

    11.2 内省

    内省就是检查实例的类型，通过创建方式来判断对象的结构和功能

    委托里使用[[Prototype]]来进行内省，可以知道这个对象是否可以调用委托对象的某个方法

    var Foo = { /* .. */ }; 
var Bar = Object.create( Foo ); 
Bar... 
var b1 = Object.create( Bar );

//内省
// FOO和BAR相互关联
Foo.isPrototypeOf( Bar ); // true 
Object.getPrototypeOf( Bar ) === Foo; // true 

// 让b1关联到Bar和Foo
Foo.isPrototypeOf( b1 ); // true 
Bar.isPrototypeOf( b1 ); // true 
Object.getPrototypeOf( b1 ) === Bar; // true
Object.getPrototypeOf(b1)===Foo //false

    canvas

    context的默认单位都是像素

    一、变形操作：变形后进行重绘坐标不变

    1.ctx.translate：实现平移，然后重绘，但是坐标不变。相对上一次的位置进行移动。**移动的是整个坐标系，之后的坐标都是相对新坐标系的值**

    ctx.fillRect(30,30,50,50)
ctx.translate(50,50)
ctx.fillRect(30,30,50,50)//重绘

    默认情况下，canvas会把所有绘制的图形保留下来。如果不想保留之前绘制的图形，需要在重绘前清空整个canvas，

    cxt.cleraRect(0,0,canvas.width,canvas.height)
//用于清空整个canvas

    2.ctx.scale(1.5,1.5):放大x轴和y轴，但是也会把左上角的坐标、线条宽度放大或缩小相应的倍数。

          context.fillStyle = 'red'
      context.fillRect(30, 30, 50, 50)
      context.scale(2, 2)
      context.translate(-20, -20)//30-2*30=-30
      context.fillStyle = 'blue'
      context.fillRect(30, 30, 50, 50)

    3.ctx.rotate(angle):[-PI/2,PI/2]，默认以原点为旋转中心，可以先translate再旋转来修改旋转中心。**顺时针为正，逆时针为负**

    4.ctx.transform()和ctx.setTransform()

    ctx.transform()//会在上一次变换后的基础上进行变换
ctx.setTransform()//先把图像重置为原始位置，再进行变换

    二、canvas路径

    函数名 描述
    beginpath 结束路径，开始新的路径，可以使用不同的绘制样式
    closePath 关闭路径，将同一个路径的起点和终点连接起来，形成封闭的路径，不一定要和beginPath配对使用。用line来封闭图形会在lineWidth很大时出现缺口，应该在stroke()之前使用closePath来关闭
    isPointInPath 判断某个点是否在某个图形内，只支持rect方法，不支持strokeRect和fillRect

    Canvas是基于状态绘制图形的，每一个绘制（stroke()和fill()），canvas会检测整个程序定义的所有状态，比如fillstyle、linewidth等。当状态值没有被改变时，Canvas就会使用最初的值，如果改变了但是没有使用beginPath()则所有的绘制都是在同一路径上，后面的绘制样式会覆盖前面的

          context.moveTo(0, 0)
      context.lineTo(100, 100)
      context.strokeStyle = 'red'
      context.stroke()

      context.moveTo(100, 0)
      context.lineTo(200, 200)
      context.strokeStyle = 'blue'
      context.stroke()
      //最终两条直线的颜色都是蓝色，因为它们是同一路径

    **如果采用了beginPath()，则开始了一个新的路径**，在不同路径里可以使用不同的绘制样式

          context.beginPath()//开始新路径
      context.moveTo(0, 0)
      context.lineTo(100, 100)
      context.strokeStyle = 'red'
      context.stroke()

      context.beginPath()//开始新路径
      context.moveTo(100, 0)
      context.lineTo(200, 200)
      context.strokeStyle = 'blue'
      context.stroke()

    三、canvas状态

    两个操作状态的方法：**save()和restore()**。

    注：保存状态只是保存的context当时的一些绘制属性，但是上一次的最后一个绘制结束点对当前还是会有影响，需要结合context.beginPath()来消除

    save：用于保存当前状态，restore用户恢复之前保存的状态，一般情况下是成对使用的。

    使用场景：图形或图片的剪切、图形或图片的变形、绘制样式发生改变。

    可以保存的状态：剪切状态、变形状态（一般指变换矩阵）、绘图状态，不能保存路径状态和图形

    clip():将一个基本图形变成裁剪区域，后面绘制的图形如果超出了这个剪切区域，则超出的部分不会被显示。该方法不支持strokeRect()和fillRect()，需要用rect()代替。

    如果后面需要**取消剪切区域，可通过save()和restore()方法。如果不使用这两个方法，即使清空画布，之前的剪切区域还是存在的。**

          let canvas = document.getElementById('canvas')
      let context = canvas.getContext('2d')

      context.save()//保存状态
      context.beginPath()
      context.arc(70, 70, 50, 0, (360 * Math.PI) / 180, true)
      context.closePath()
      context.stroke()
      context.clip()//绘制圆形剪切区域

      var image = new Image()
      image.src = '1.jpg'
      image.onload = function() {
        context.drawImage(image, 10, 20)
      }

      var button = document.getElementById('button')
      button.onclick = function() {
        context.restore() //恢复之前保存的状态，其实是空状态（没有剪切区域），但是之前绘制的圆还是在的
        context.clearRect(0, 0, canvas.width, canvas.height) //清空画布，清除之前的圆

        var image = new Image()
        image.src = '2.jpg'
        image.onload = function() {
          context.drawImage(image, 10, 20)
        }
      }

    四、canvas属性

    属性 描述
    width 宽
    height 高
    getContext 获取上下文环境
    toDataURL 把画布转成图片保存
    globalAlpha 设置画布的透明度
    globalCompositeOperation 交叉图形的显示方式

    正常情况下，浏览器按照图形的绘制顺序依次显示每个图形，后面绘制的会覆盖前面绘制的，更改交叉图形显示方式可以使用globalCompositeOperation属性

    五、canvas事件

    1、鼠标事件

    mousedown、mouseup、mousemove

    2、键盘事件

    canvas本身不支持键盘事件，可以借助window对象来实现，keyup和keydown

    3、循环事件

    canvas中动画循环中的注意事项：

    1. 对于需要不断改变的变量，一般在动画循环之前定义
    2. 对于需要不断改变的变量，一般在动画循环中图形绘制之后才递增或递减

    3.1requestAnimationFrame设置动画

    不需要手动设置间隔时间，会根据浏览器绘制的帧率自动进行调整，效果上比较顺滑。

            (function drawFrame() {
          window.requestAnimationFrame(drawFrame);
          cxt.clearRect(0, 0, cnv.width, cnv.height);
            //绘制相关图形
              //
        })();

    3.2setInterval实现动画

    存在一定的性能问题，可以一次全部重绘来减小卡顿的情况。

    setInterval(function(){    //绘制图形},10)

    3.3双缓冲实现

    先将所有内容绘制到缓冲canvas上，然后把缓冲canvas绘制到DOM元素canvas上，可以解决闪屏

        var canvas = document.getElementById('air-speed')
    var context = canvas.getContext('2d')

    var bufferCanvas = document.createElement('canvas')
    bufferCanvas.width = canvas.width
    bufferCanvas.height = canvas.height
    var bufferContext = bufferCanvas.getContext('2d')

    bufferContext.fillStyle = "red"
    bufferContext.fillRect(0, 0, 100, 100)

    context.drawImage(bufferCanvas, 0, 0,                             bufferCanvas.width, bufferCanvas.height, 0, 0, canvas.width, canvas.height)

    六、物理动画

    1.三角函数

    用途：求两点之间的距离、圆周运动、波形运动

    注：Math.atan2(y,x)可以求出两条边的角度并准确判断该度数对应的是哪一个夹角，y和x要区分正负

    1.1 两点之间距离

    $$
    d=Math.sqrt(pow(x1-x2,2)+pow(y1-y1,2))
    $$

    1.2圆周运动

    1.2.1 正圆运动

    圆上某个点的坐标为
    $$
    x=centerX + Math.cos(angle) * radius;
    $$

    $$
    y=centerY + Math.sin(angle) * radius;
    $$

    其中centerX、centerY为圆心坐标，angle是弧度制角度，radius为圆半径

    1.2.2 椭圆运动

    椭圆上某个点的坐标为
    $$
    x=centerX + Math.cos(angle) * radiusX;
    $$

    $$
    y=centerY + Math.sin(angle) * radiusY;
    $$

    其中radiusX为椭圆X轴方向的半径，radiusY为Y轴半径

    1.3波形运动

    1.3.1作用于X轴

    当正弦函数作用于物体的X坐标，物体会左右摇摆
    $$
    x = centerX + Math.sin(angle) * range;
    $$

    $$
    angle+=speed
    $$

    其中centerX为物体中心坐标，angle是弧度制角度，range是振幅，speed是角度变化大小

    1.3.2作用于Y轴

    物体的运动轨迹刚好就是sin函数的波形
    $$
    y = centerY + Math.sin(angle) * range;
    $$

    $$
    angle+=speed
    $$

    $$
    x+=1
    $$

    注：需要配合x改变才能是函数波形

    1.3.3 作用于缩放属性scaleX或scaleY

    物体会不断地放大然后缩小，呈现一种脉冲动画的效果

    scaleX = 1 + Math.sin(angle) * range;
scaleY = 1 + Math.sin(angle) * range;
angle+=speed

    2.匀速运动

    每次循环改变量都相同，vx为x方向的速度，vy为y方向的速度。vx为负，向X负轴运动；为正时，向x正轴运动

    x += vx;//或y+=vy;

    如果已知任意一个方向angle的速度为speed，则对它进行分解有

                //速度方向与X轴正方向角度为angle
            var vx = speed * Math.cos(angle * Math.PI / 180);
            var vy = speed * Math.sin(angle * Math.PI / 180);

    3.加速运动

    3.1匀加速

    vx+=ax
vy+=ay//ax为x方向加速度，为0是匀速。加速度方向与速度方向相同则是匀加速x+=vxy+=vy

    如果已知某个方向angle的加速度为a，则对加速度分解

    //对加速度分解
ax=a * Math.cos(angle * Math.PI / 180);
ay=a * Math.sin(angle * Math.PI / 180);

vx+=ax
vy+=ay
//ax为x方向加速度，为0是匀速。加速度方向与速度方向相同则是匀加速
x+=vx
y+=vy

    3.2 重力加速度

    对于重力引起的运动，可以看成是沿着Y轴正方向的加速运动

    对于小球地面反弹的情况，需要将运动速度乘以反弹系数（-1,0）。

    3.3 摩擦力

    只能将物体的速度降至0，但不能让物体向相反的方向运动

    vx *= friction;//friction为摩擦系数，[0,1]x  +=vx

    4.边界检测

    先判断边界，再进行绘制

    4.1 边界限制

    通过检测边界来限制物体的运动范围，使其无法超出这个运动范围

                //定义边界检测函数
            function checkBorder() {
                //当小球碰到上边界时
                if (ball.y < ball.radius) {
                    ball.y = ball.radius;
                    //当小球碰到下边界时
                } else if (ball.y > cnv.height - ball.radius) {
                    ball.y = cnv.height - ball.radius;
                }
                //当小球碰到左边界时
                if (ball.x < ball.radius) {
                    ball.x = ball.radius;
                    //当小球碰到右边界时
                } else if (ball.x > cnv.width - ball.radius) {
                    ball.x = cnv.width - ball.radius;
                }
            }

    4.2 边界环绕

    物体从左边界消失后从右边界出来；从上边界消失，从下边界出

                    //完全超出边界
                //检测上边界
                if (ball.y < -ball.radius) {
                  ball.y = cnv.height + ball.radius;
                }
                //检测下边界
                if (ball.y > cnv.height + ball.radius) {
                  ball.y = -ball.radius;
                }
                   //检测左边界
                if (ball.x < -ball.radius) {
                  ball.x = cnv.width + ball.radius;
                }
                //检测右边界
                if (ball.x > cnv.width + ball.radius) {
                  ball.x = -ball.radius;
                }

    4.3 边界生成

    当物体完全超出边界之后，会在最开始的位置重新生成。可以源源不断地为canvas提供运动物体

                        //边界检测，使得小球完全移出画布后会在中心位置重新生成
                    if (ball.x < -ball.radius ||
                      ball.x > cnv.width + ball.radius ||
                      ball.y < -ball.radius ||
                      ball.y > cnv.height + ball.radius) 
                    {
                        ball.x = cnv.width / 2;
                        ball.y = cnv.height / 2;
                        ball.vx = Math.random() * 2 - 1;
                        ball.vy = Math.random() * 2 - 1;
                    }

    4.4 边界反弹

    碰到边界后，保持位置不变，改变速度向量。碰到左边界或者右边界，就对vx取反，vy不变；碰到上边界或下边界，对vy取反，vx不变。

    //边界检测
                //碰到左边界
                if (ball.x < ball.radius) {
                    ball.x = ball.radius;
                    vx = -vx;
                    //碰到右边界
                } else if (ball.x > canvas.width - ball.radius) {
                    ball.x = canvas.width - ball.radius;
                    vx = -vx;
                }
                //碰到上边界
                if (ball.y < ball.radius) {
                    ball.y = ball.radius;
                    vy = -vy;
                    //碰到下边界
                } else if (ball.y > canvas.height - ball.radius) {
                    ball.y = canvas.height - ball.radius;
                    vy = -vy;
                }

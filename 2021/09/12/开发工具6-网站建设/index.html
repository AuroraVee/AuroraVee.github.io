    网站建设基础

    1.什么是WWW？

    万维网，W3C，制定了Web标准，包括HTML、css、xml的标准

    2.万维网如何工作？

    Web页面存储在Web服务器上，阅读网页的计算机叫做Web客户端，客户端用于访问页面的程序叫做web浏览器

    浏览器访问页面时，需要通过HTTP向服务器请求，如果请求的HTML中包含服务器脚本，需要先执行脚本再返回纯HTML文件，对HTML解析渲染。浏览器显示的内容都有 HTML、XML、GIF、Flash 等，浏览器是通过 MIME Type 区分它们，决定用什么内容什么形式来显示。

    3.Web服务器

    Apache服务器，IIS服务器，Nginx服务器

    服务器就是能够提供服务的机器，这取决于机器上所安装的软件(服务软件)，比如：Web服务器提供Web服务（网站访问），就需要Web服务软件，如Apache、tomact等。

    4.脚本

    客户端脚本：对浏览器的行为，JavaScript，其标准是ECMASCRIPT

    服务器脚本：对服务器的行为，ASP、PHP，涉及到数据库的访问，对浏览器请求进行响应

    5.HTML、css、Js、xml

    html：定义网页的内容

    css：设置网页的样式、外观

    js：对浏览器的行为，HTML表单数据验证，与用户交互的事件等

    xml：存储、传输数据，结合JS通过ajax与服务器交换数据

    6.搜索引擎优化SEO

    网站上进行数据检索依赖的是搜索引擎（比如百度、谷歌），采用一定策略从互联网上搜集信息，然后展示。

    搜索引擎包括全文索引、目录索引、元搜索引擎、垂直搜索引擎、集合式搜索引擎、门户搜索引擎与免费链接列表等。

    全文索引：首先是数据库中数据的搜集，搜索引擎的自动信息搜集功能分两种。一种是定期搜索，即每隔一段时间（比如Google一般是28天），搜索引擎主动派出“蜘蛛”程序，对一定IP地址范围内的互联网网站进行检索，一旦发现新的网站，它会自动提取网站的信息和网址加入自己的数据库。另一种是提交网站搜索，即网站拥有者主动向搜索引擎提交网址，（https://www.runoob.com/web/web-search.html）它在一定时间内（2天到数月不等）定向向你的网站派出“蜘蛛”程序，扫描你的网站并将有关信息存入数据库，以备用户查询。

    当用户以关键词查找信息时，搜索引擎会在数据库中进行搜寻，如果找到与用户要求内容相符的网站，便采用特殊的算法——通常根据网页中关键词的匹配程度、出现的位置、频次、链接质量——计算出各网页的相关度及排名等级，然后根据关联度高低，按顺序将这些网页链接返回给用户。

    8.网站主机

    8.1 原理

    如果您的 PC 连入网络的话，您也可以把它当作一台 web 服务器（IP地址是临时的）。

    要想让别人看到您的页面，就必须对网站进行发布，一般需要把网站拷贝到ISP提供的web主机。ISP 指的是 Internet 服务提供商，可提供Internet 服务（网站主机就是其中的一种），网站主机服务会通过DNS将域名（主域名：域名注册机构进行注册，子域名www等可由DNS服务器或者网站主机提供商生成）和IP地址相互映射存放在公共服务器上（即需要向ISP申请固定的IP地址，购买后可以在服务器上存放其他人需要访问的资源）。

    8.2**私有IP地址（局域网、内网）：**

    A类地址：10.0.0.0——10.255.255.255

    B类地址：172.16.0.0—-172.31.255.255

    C类地址：192.168.0.0—192.168.255.255

    内网的计算机以NAT（网络地址转换）协议，通过一个公共的网关访问Internet。内网的计算机可向Internet上的其他计算机发送连接请求，但Internet上其他的计算机无法向内网的计算机发送连接请求。

    外网IP标识你在整个互联网的地址
    内网IP标识你在你所在的局域网里面的地址

    8.3性能参数

    磁盘空间、月流量（数据传输量）、连接速度

    8.4 网站主机类型

    免费主机、虚拟（分享的）主机或独享主机。

  console.log(req.method); // 获取请求方法
  console.log(req.url); // 获取请求路径（包含查询字符串）
  console.log(req.httpVersion); // 获取 HTTP 协议版本
  console.log(req.headers); // 获取请求头（对象）
   
    
  // 设置响应头（过去的用法），不能多次调用，见到要认识
  res.writeHead(200, { "Content-Type": "text", a: "hello world" });

  // 设置响应头（现在的用法，常用），可以多次调用，每次设置一个响应头
  res.setHeader("Content-Type", "text");

  // 设置状态码，不设置默认为 200
  res.statusCode = 200;

  // 不发送 Date（日期）响应头
  res.sendDate = false;

  // 返回内容
  res.write("hello world"); // 不会关闭连接
  res.end("hello world"); // 将内容返回后关闭连接
});

server.listen(3000, function() {
  console.log("server start 3000");
});

    11.ARP协议

    网络层、传输层、应用层只能标识IP地址，而数据链路层只标识MAC地址，所以需要ARP协议将IP地址解析成MAC地址，然后通过MAC进行中转

    ARP缓存表保存了IP地址和MAC地址的映射关系，每一个记录称作ARP表项，当需要发送数据时，主机会根据目标IP地址在缓存表中查找对应的MAC地址，然后通过网卡将数据发送出去。每个表项都有生存时间TTL，时间到了会自动从缓存表中删除。

    局域网通信：

    查询缓存表时如果没有目标IP地址的映射，会在局域网上广播ARP请求，如果某个主机的IP地址和目标IP地址相同就会返回自身的MAC地址，当电脑收到ARP应答后会建立ARP表项，重新将数据发送出去。

    公网通信

    需要先将数据发送到网关，网关会发送ARP请求，得到MAC地址后建立表项，完成通信

    16.https

    通过SSL（安全套接层）或者TLS（安全传输层协议）建立安全的通信线路，另外还可以为服务器和客户端向第三方机构申请证书，用于验证通信方的真实意图。

    http不能验证通信方的身份，会遭受DOS攻击（拒绝服务攻击）

    http不能确保信息传输前后的准确性，会遭受MITM攻击（中间人攻击）

    17.cookie存储客户端状态和jwt存储区别

    17.1cookie存储

    客户端发出http请求，服务器验证后会生成一个sessionID，并存在服务器数据库，服务器在响应添加字段Set-Cookie返回给客户端；客户端把会话ID存储在Cookie中，以后请求时在首部次数携带上Cookie，然后与服务器存储的sessionID进行匹配。一旦用户登出，则 session 在客户端和服务器端都被销毁

    17.2 JSON Web Token

    http://www.ruanyifeng.com/blog/2018/07/json_web_token-tutorial.html

    服务器生成JWT之后，不需要将JWT存储在服务器上，JWT的payload中存有用户的登录状态、权限、过期时间等，可以在下次请求时解码，得到payload验证即可。

    1. 用户输入登录信息
    2. 服务器判断登录信息正确，返回一个 token
    3. token 存储在客户端，大多数通常在 local storage（防御CSRF攻击），但是也可以存储在 session storage 或者 cookie 中。
    4. 接着客户端发起请求的时候将 token 放进 Authorization header，或者同样可以通过上面的方式。
    5. 服务器端解码 JWT 然后验证 token，如果 token 有效，则处理该请求。
    6. 一旦用户登出，token 在客户端被销毁，不需要经过服务器端。

    18.http2.0

    1.与http1.1的API相同

    2.以SPDY为基础

    1. 二进制分帧，首部放在headers帧，报文主体放在Data帧
    2. 多路复用，多个流并行，最后根据流标识符进行组装
    3. 头部压缩：通讯双方各自缓存一份头部字段表，用索引代替首部（HPACK算法）
    4. 请求优先级：优先级最高HTML，高CSS文件，中JS文件，低图片
    5. 服务器推送：服务端需要主动推送某个资源时会告诉客户端，遵循同源的情况下，不同页面之间可以共享缓存资源

    服务器推送：

    http://www.ruanyifeng.com/blog/2018/03/http2_server_push.html

    传统网页需要先请求html，得到响应后再请求CSS等文件资源。服务器推送，可以实现请求html的时候，一次性把HTML中相关的所有资源作为响应返回给服务器，减少了请求次数。

    19.CGI和Serverlet

    CGI：通用网关接口，根据请求信息动态产生回应内容的技术，主要用Perl、Shell Script或C编写。对于每个请求，都要产生一个新的CGI程序（进程）进行处理，占用很多内存。

    Java Servlet要求必须运行在Web服务器当中，在服务器上创建动态的内容，每个请求都是单独启动一个线程，所有的线程都和web服务器在一个进程中

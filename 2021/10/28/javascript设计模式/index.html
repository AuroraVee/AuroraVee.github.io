    javascript设计模式

    该笔记是针对书本《javascript设计模式》写的，边看边记

    1.接口

    从Java的角度讲，一个接口会定义好所有需要实现的方法，以及这些方法所需要的参数和参数类型。Java编译器可以自动检测某个类A声明实现接口B时是否真的实现了那些必要的方法。但是JavaScript不能自动识别，需要手动写接口的定义以及对实现接口的类进行检查。

    JavaScript可以通过注释、属性检测、鸭式辩型来模仿接口的实现。将注释和鸭式辩型结合起来既可以有文档的注释也可以进行检测，是比较好的一种方式。

    /*
声明两个接口应该实现的方法
Interface Composite{
    function add();
    function remove();
    function getChild();
}

Interface Form{
    function save();
}
*/

let Composite=new Interface("Composite",["add","remove","getChild"])
let Form=new Interface("Form",["save"])

let addForm(instance){
    Interface.ensureImplements(instance,Composite,Form)
    //do someting
    //确保某个实例实现了Composite和Form接口时再去进行相应的操作
}

    functiom Interface(name,methods){
    this.name=name
    this.methods=methods
}

Interface.ensureImplements=function(object){
    for(let i=1;i<arguments.length;i++){
       let interfaceObj= arguments[i]
       if(interfaceObj.constructor!==Interface ){
           //报错
        }
        for(let j=0;j<interfaceObj.methods.length;j++){//这个接口需要实现的所有方法
            let method=interfaceObj.methods[j]
            if(!object[method] || typeof method!=='function'){
                //报错
            }
        }
    }
}

    为所依赖的每一个API都创建Interface对象，并对接受到的每个对象进行检测，来确保正确实现了接口，然后再去调用执行相应功能。

    相比使用构造器进行参数检查，这种接口的检测方式不限制传进来的对象一定要是某个类的实例

    2.封装与信息隐藏

    信息隐藏是目的，封装是实现该目的的技术手段。

    理想的类是将内部属性和方法隐藏起来，避免外部随意更改，只提供给外部接口所指定的方法。外部也只能通过指定的存取方法操作内部属性。需要通过封装的方式，提供公用、私用、特权方法。

    2.1 方法分类

    从是否可以操作内部属性分为：

    1. 公用方法：放在类的外部，不能直接操作私有数据，所有实例都可以直接使用该方法；一般放在prototype上
    2. 私用方法：放在类的内部，只能在内部使用，可以操作私有数据；
    3. 特权方法：放在类的内部，类的外部和内部都可以使用该方法，可以操作私有数据，一般通过this绑定到对象上

    从所属对象分：

    1. 静态方法和静态属性：直接通过类访问，所有实例共用一份；
    2. 普通方法和属性：通过实例对象访问，每个实例对象都会各自保存一份副本；内存占用多

    2.2 封装方法

    2.2.1 门户大开型

    即将有this操作的普通函数作为构造函数（只含有数据成员）。所有的属性都是通过this关键字创建，方法放在构造函数的prototype属性上，这些属性和方法都是公用的。

    let book=function(isbn,name,author){
    this.isbn=isbn;
    this.name=name;
    this.author=author;
}

book.prototype={
    display:function(){
        
    };
    //...
}

    程序员创建实例对象后可以随意的修改，比如let obj=new book("1234567890","test","wjx");obj.isbn=" "。就算在prototype上为每个数据属性都提供get和set方法，也不能阻止程序员的修改。

    2.2.2 用命名规范区别私有成员

    默认规定以下划线开头的方法和属性是私有的。它只是种约定，如果不准守也可以随意修改数据。

    let book=function(isbn){
    this.setIsbn(isbn);
}

book.prototype={
    display:function(){
        
    };
    getIsbn(){
        return this._isbn;
    };
    setIsbn(isbn){
        this._isbn=isbn
    }
    //...
}

    私有属性就是指定只能在内部使用，这跟JavaScript的作用域是一样的。另外，在JavaScript中定义在函数里的内嵌函数可以一直访问同一个作用域下的成员。

    为了创建私用成员，需要在构造函数的作用域里声明所有的私有变量，这些变量可以被该作用域中的所有函数访问，包括特权方法。

    3.继承

    3.1 类式继承

    用函数**声明类**，通过关键字new创建实例

    function extend(subClass, superClass) {
  var F = function() {};
  F.prototype = superClass.prototype;//避免创建父类的新实例
  subClass.prototype = new F();//子类的prototype等于父类的实例
  subClass.prototype.constructor = subClass;

  subClass.superclass = superClass.prototype;
  if(superClass.prototype.constructor == Object.prototype.constructor) {
    superClass.prototype.constructor = superClass; //确保父类的constructor属性就是自己
  }
}

/* Class Person. */

function Person(name) {
  this.name = name;
}

Person.prototype.getName = function() {
  return this.name;
}

/* Class Author. */

function Author(name, books) {
  //Person.call(this, name);
  Author.superclass.constructor.call(this,name)//避免直接将父类对象固定
  this.books = books;
}

extend(Author, Person);

Author.prototype.getBooks = function() {
  return this.books;
};

//重写getName函数
Author.prototype.getName = function() {
  var name = Author.superclass.getName.call(this);
  return name + ', Author of ' + this.getBooks().join(', ');
};

    3.2 原型链继承

    直接创建对象字面量，这个对象可以被新的对象通过原型链访问，被称为**原型对象。一个对象只能有一个直接的原型对象**

    原型对象和子对象通过一个clone函数来连接：

    /* Clone function. */

function clone(object) {
    function F() {}
    F.prototype = object;
    return new F;
}

    //原型对象，提供了属性和方法的默认值
var Person = {
  name: 'default name',
  getName: function() {
    return this.name;
  }
};

//创建一个类似于Person的新对象，不并再创建一个子类，
//可以重定义克隆下来的方法和属性，也可以直接修改默认值
var Author = clone(Person);//直接通过创建一个空对象，然后让它的prototype指向Person即可
Author.books = []; // 为Author子对象创建了books属性，getBooks方法
Author.getBooks = function() {
  return this.books;
}

    原型链搜索时，优先搜索子对象自身是否具有某个属性和方法，如果没有再在原型对象上搜索，然后一直向上直到结束。要注意新创建的**子对象自身是否有和原型对象一样的属性或方法，要不然可能会直接修改原型对象**，这将影响到其他的子对象。

    //eg.1
var authorClone = clone(Author);
alert(authorClone.name); // 实际访问的是Person.name，‘default name’

authorClone.name = 'new name';
alert(authorClone.name); //实际访问的是authorClone.name，‘new name’

//eg.2
authorClone.books.push('new book'); //实际修改的是Author.books
//直接修改原型对象，会影响后面所有继承原型对象的子对象

    hasOwnProperty可以用来区分属性和方法是自身的还是继承来的，但是为了能更好地避免直接修改原型对象，可以通过以下的工厂方法来创建子对象：

    var CompoundObject = {};
CompoundObject.string1 = 'default value',
CompoundObject.createChildObject = function() {//该方法用于创建具有和原型对象相同的数据结构，对象之间没有引用
  return {
    bool: true,
    num: 10
  }
};
CompoundObject.childObject = CompoundObject.createChildObject();

var compoundObjectClone = clone(CompoundObject);
compoundObjectClone.childObject = CompoundObject.createChildObject();
compoundObjectClone.childObject.num = 5;

    3.3 比较

    方式 特点
    类式继承 用类来定义对象的结构，这种方法创建出的对象属性都是各个保留副本，方法共用一份
    原型链继承 直接创建对象子面量，直接克隆时所有子对象都是共享原型对象的数据，更节约内存

    3.4 继承与封装

    方式 特点
    门户大开型 所有成员都是公开的，可以用下划线将私有成员隐藏
    用闭包实现 共用方法和特权方法是公开的，子类只能通过特权方法访问父类的私有成员

    3.5 掺元类mixin class

    一个包含通用方法的类被叫作掺元类，它通常不用于实例化或直接调用，只是向其他类提供自己的方法

    C++或者Python等语言都支持一个子类同时继承多个父类，但是**JavaScript只支持一个对象有一个直接的原型对象，这时可以通过一个类用多个掺元类扩充实现多继承。**

    //补充方法
//将掺元类中的所有方法都复制到目标类中
function augment(receivingClass, givingClass) {
  if(arguments[2]) { // Only give certain methods.
    for(var i = 2, len = arguments.length; i < len; i++) {
      receivingClass.prototype[arguments[i]] = givingClass.prototype[arguments[i]];
    }
  } 
  else { // Give all methods.
    for(methodName in givingClass.prototype) { 
      if(!receivingClass.prototype[methodName]) {
        receivingClass.prototype[methodName] = givingClass.prototype[methodName];
      }
    }
  }
}

    //掺元类
/* Mixin class. */

var Mixin = function() {};
Mixin.prototype = {
  serialize: function() {
    var output = [];
    for(key in this) {
      output.push(key + ': ' + this[key]);
    }
    return output.join(', ');
  }
};

augment(Author, Mixin);

var author = new Author('Ross Harmes', ['JavaScript Design Patterns']);
var serializedString = author.serialize();

    4.单体模式

    单体是一个用来划分命名空间并且将一些相关的方法和属性融合在一起的对象（普通的对象字面量），它如果可以被实例化，只可以实例化一次。

    单体模式是JavaScript最流行、使用最广泛的设计模式之一。

    4.1 划分命名空间

    将属性和方法作为对象的一部分，可以避免都暴露在全局环境中，可以减少冲突。

    /* GiantCorp namespace. */
var GiantCorp = {};//用于存储所有代码的对象

GiantCorp.Common = { //
  // A singleton with common methods used by all objects and modules.
};

GiantCorp.ErrorCodes = {
  // An object literal used to store data.
};

GiantCorp.PageHandler = {
  // A singleton with page specific methods and attributes.
};

    4.2 用于特定网页专用代码的包装器的单体

    4.2.1 简单的情况

    这个单体中一般包含各个方法公用的数据属性、公用方法、初始化方法。

    各个方法之间互相调用，如果采用this来调用需要注意：**当某个方法用作事件的回调函数时，this将不再指向当前的单体对象。**

    /* Generic Page Object. */

Namespace.PageName = {

  // Page constants.
  CONSTANT_1: true,
  CONSTANT_2: 10,

  // Page methods.
  method1: function() {

  },
  method2: function() {

  },

  // Initialization method.
  init: function() {

  }
}

// Invoke the initialization method after the page loads.
addLoadEvent(Namespace.PageName.init);

    4.2.2 有私有成员的单体

    4.2.2.1 用下划线表示私有

    将所有的私有属性和私有方法都用下划线_开头，以此来告诫其他程序员这是内部使用的。

    GiantCorp.DataParser = {
  // Private methods.
  _stripWhitespace: function(str) {
    return str.replace(/\s+/, '');
  },
  _stringSplit: function(str, delimiter) {
    return str.split(delimiter);
  },
  
  // Public method.
  stringToArray: function(str, delimiter, stripWS) {
    if(stripWS) {
      str = this._stripWhitespace(str); //this调用内部方法
    }
    var outputArray = this._stringSplit(str, delimiter);//this调用内部方法
    return outputArray;
  }
};
    4.2.2.2 使用闭包实现私有

    在第3节使用闭包实现封装时，我们采用在构造函数里声明所有的私有变量和方法，然后采用this将特权方法提供给外部使用。这种方式，每次创建一个实例，所有的方法和属性都会各自保存一份，十分浪费内存。

    但是由于单体模式只有一个实例，每个方法和属性都只会创建一次，所以还是可以把所有的私有成员写在构造函数里。这种模式又称为模块模式

    //用立即执行的函数来创建单体
//这个函数创建了一个用来添加私有成员的闭包，在立即函数执行结束后，由于单体对象和私有成员同处于一个作用域，所有可以通过单体对象访问私有成员

GiantCorp.DataParser = (function() {
  // Private attributes.
  var whitespaceRegex = /\s+/;
  
  // Private methods.
  function stripWhitespace(str) {
    return str.replace(whitespaceRegex, '');
  }
  function stringSplit(str, delimiter) {
    return str.split(delimiter);
  }
  
  // Everything returned in the object literal is public, but can access the
  // members in the closure created above.
  return { 
    // Public method.
    stringToArray: function(str, delimiter, stripWS) {
      if(stripWS) {
        str = stripWhitespace(str);//访问私有成员
      }
      var outputArray = stringSplit(str, delimiter);
      return outputArray;//访问私有成员
    }
  };
})(); // Invoke the function and assign the returned object literal to 
     // GiantCorp.DataParser.

    4.3 惰性实例化

    上面4.2.2.2中使用闭包实现私有时是借助立即执行函数来完成的，这种方式会在脚本加载时就立即实例化。这对于那些被用作命名控件、特定网页专用代码包装器、组织相关使用方法的单体可以使用，但是对于某些加载了大量数据的单体，我们可以将实例化推迟到使用它的时候。而这种技术叫作惰性加载。

    具体实现方法就是，将之前的立即执行函数返回的单体对象修改为一个包含控制单体实例化的普通对象{getInstance}，然后再在需要时手动调用普通对象的方法完成单体对象的实例化

    MyNamespace.Singleton = (function() {
  
  var uniqueInstance; // Private attribute that holds the single instance.
  
  function constructor() { // All of the normal singleton code goes here.
    ...
  }
  
  return {
    getInstance: function() {
      if(!uniqueInstance) { // Instantiate only if the instance doesn't exist.
        uniqueInstance = constructor();
      }
      return uniqueInstance;
    }
  }
})();
    
//在需要时调用，进行单体对象的实例化
 MyNamespace.Singleton.getInstance() 
//访问单体对象的方法
MyNamespace.Singleton.getInstance() .method()

    4.4 分支

    分支：把浏览器之间的差异封装到能够在运行期间设置的动态方法中。这种技术可以减少浏览器嗅探和对象检测的次数，加快调用效率。

    分支**适用于任何需要在运行时才能确定的具体实现的情况**。

    4.4.1 通过单体对象实现

    前面在用闭包创建私有成员时，就是在运行时创建的。我们可以针对不同的浏览器设置不同的单体对象，然后在立即执行函数里根据运行时的状态返回特定的对象。

    由于需要针对不同的情况提供不同的单体对象，这将会占用更多的内存。需要在内存和计算时间上做一定的平衡。

    /* Branching Singleton (skeleton). */

MyNamespace.Singleton = (function() {
  var objectA = {
    method1: function() {
      ...
    },
    method2: function() {
      ...
    }
  };
  var objectB = {
    method1: function() {
      ...
    },
    method2: function() {
      ...
    }
  };
    //两个单体对象的接口是一致的，所以对于外部调用没有区别，不同的是内部的实现方法
  return (someCondition) ? objectA : objectB;
})();

    4.4.2 使用惰性函数实现

    当我们每次都需要进行条件判断，其实只需要判断一次，接下来的使用方式都不会发生改变的时候，想想是否可以考虑使用惰性函数。这种技术又叫作memorize

    //在判断一次后，就立即修改函数，后续就不需要再次试探
function addEvent (type, el, fn) {
    if (window.addEventListener) {
        addEvent = function (type, el, fn) {
            el.addEventListener(type, fn, false);
        }
    }
    else if(window.attachEvent){
        addEvent = function (type, el, fn) {
            el.attachEvent('on' + type, fn);
        }
    }
    
    addEvent(type, el, fn) //调用一次，给目标元素绑定好事件
}

    4.4.3 用工厂模式实现

    用工厂函数根据不同的浏览器情况创建不同的实例对象

    /* AjaxHandler interface. */

var AjaxHandler = new Interface('AjaxHandler', ['request', 'createXhrObject']);

/* SimpleHandler class. */

var SimpleHandler = function() {}; // implements AjaxHandler
SimpleHandler.prototype = {
  request: function(method, url, callback, postVars) {//请求的逻辑过程是固定的
    var xhr = this.createXhrObject();//不同的是创建实例对象的方法
    xhr.onreadystatechange = function() {
      if(xhr.readyState !== 4) return;
      (xhr.status === 200) ? 
        callback.success(xhr.responseText, xhr.responseXML) : 
        callback.failure(xhr.status);
    };
    xhr.open(method, url, true);
    if(method !== 'POST') postVars = null;
    xhr.send(postVars);
  },
  createXhrObject: function() { // Factory method.工厂方法
    var methods = [
      function() { return new XMLHttpRequest(); },
      function() { return new ActiveXObject('Msxml2.XMLHTTP'); },
      function() { return new ActiveXObject('Microsoft.XMLHTTP'); }
    ];
    
    for(var i = 0, len = methods.length; i < len; i++) {//首次会遍历所有可用的方法
      try {
        methods[i]();
      }
      catch(e) {
        continue;
      }
      // If we reach this point, method[i] worked.
      this.createXhrObject = methods[i]; // Memoize the method.一旦检测到方法可用就直接修改掉，后续不需要再检测
      return methods[i];
    }
    
    // If we reach this point, none of the methods worked.
    throw new Error('SimpleHandler: Could not create an XHR object.');
  } 
};

  success: function(responseText) { alert('Success: ' + responseText); }, 
  failure: function(statusCode) { alert('Failure: ' + statusCode); } 
};
myHandler.request('GET', 'script.php', callback);

    5.方法的链式调用

    链式调用的本质是每个方法调用后通过return this返回同一个对象的引用，然后利用对象的点操作符.不断的继续调用方法。其中链式调用的开头就是创建出一个实例对象，后续的方法都是这个对象来调用的。

    5.1 链式调用构造方法

    (function(){
    
 function _$ (els)  //构造函数
    this.elements = []
    for (var i = 0, len = els.length; i < len; ++i) {
      var element = els[i]
      if (typeof element == 'string') {
        element = document.getElementById(element)
      }
      this.elements.push(element)
    }
  }
  _$.prototype = {//实例对象可以继承的方法
    each: function (fn) {
      for (var i = 0, len = this.elements.length; i < len; ++i) {
        fn.call(this, this.elements[i])
      }
      return this //返回实例对象的引用
    },
    setStyle: function (prop, val) {
      this.each(function (el) {
        el.style[prop] = val
      })
      return this //返回实例对象的引用，其他数据怎么存
    },
    show: function () {
      var that = this
      this.each(function (el) {
        that.setStyle('display', 'block')
      })
      return this //返回实例对象的引用
    },

        }

          el.attachEvent('on' + type, fn)
        }
      }
      this.each(function (el) {
        add(el)
      })
      return this //
    }
  }

    8.组合模式

    专为**创建web上的动态用户界面**而量身定做的模式

    组合模式擅长对大批对象进行操作，它专为组织这类对象并将操作从一层次传递到下一层次而设计

    该模式的优点：

    1. 对组合对象执行的操作将会向下传递到其子对象，然后子对象也会执行同样的操作；这有助于弱化各个对象的耦合
    2. 一批对象将形成树形结构，每个组合对象都有一个获取子对象的方法；这样就可以隐藏实现细节；

    8.1 组合对象的结构

    组合对象和构成它的子对象将形成树形结构。这个结构中共有两种对象：组合对象和叶对象。叶对象是组合对象最基本的元素，也是最终各个操作的落实点。

    组合对象和叶对象只是所在层次不同，并没有任何继承关系，他们应该实现相同的接口。

    8.2 表单验证

    需求：

    1. 希望能够通过对CompositeForm执行Save操作，能自动对该表单中的所有域也执行Save操作；
    2. 有两个FieldSet可以单独控制自己的子对象执行同样的操作，不影响其他的叶对象

    通过分析，可知整个组合模式的组织结构如下图所示：

    由于每个对象都应该实现同样的接口，这里我们定义了两个接口：Composite接口方便组合对象添加、删除、获取子对象；FormItem接口方便对所有对象执行同样的操作。总共需要实现的类如下图所示。

    /* CompositeForm class. */

var CompositeForm = function(id, method, action) { // implements Composite, FormItem
  this.formComponents = [];//保存该组合对象的子对象

  this.element = document.createElement('form');
  this.element.id = id;
  this.element.method = method || 'POST';
  this.element.action = action || '#';
};

CompositeForm.prototype.add = function(child) {
  Interface.ensureImplements(child, Composite, FormItem);//确保子对象都实现了同样的接口
  this.formComponents.push(child);
  this.element.appendChild(child.getElement());//添加到DOM元素中
};

CompositeForm.prototype.remove = function(child) {
  for(var i = 0, len = this.formComponents.length; i < len; i++) {
    if(this.formComponents[i] === child) {
      this.formComponents.splice(i, 1); // Remove one element from the array at 
                                        // position i.
      break;
    }
  }
};

CompositeForm.prototype.getChild = function(i) {
  return this.formComponents[i];
};

CompositeForm.prototype.save = function() {
  for(var i = 0, len = this.formComponents.length; i < len; i++) {
    this.formComponents[i].save();//遍历该组合对象的所有子对象，执行save操作
  }
};

CompositeForm.prototype.getElement = function() { 
  return this.element; 
};

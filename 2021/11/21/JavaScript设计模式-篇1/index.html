    4.单体模式

    单体是一个用来划分命名空间并且将一些相关的方法和属性融合在一起的对象（普通的对象字面量），它如果可以被实例化，只可以实例化一次。

    单体模式是JavaScript最流行、使用最广泛的设计模式之一。

    4.1 划分命名空间

    将属性和方法作为对象的一部分，可以避免都暴露在全局环境中，可以减少冲突。

    /* GiantCorp namespace. */
var GiantCorp = {};//用于存储所有代码的对象

GiantCorp.Common = { //
  // A singleton with common methods used by all objects and modules.
};

GiantCorp.ErrorCodes = {
  // An object literal used to store data.
};

GiantCorp.PageHandler = {
  // A singleton with page specific methods and attributes.
};

    4.2 用于特定网页专用代码的包装器的单体

    4.2.1 简单的情况

    这个单体中一般包含各个方法公用的数据属性、公用方法、初始化方法。

    各个方法之间互相调用，如果采用this来调用需要注意：**当某个方法用作事件的回调函数时，this将不再指向当前的单体对象。**

    /* Generic Page Object. */

Namespace.PageName = {

  // Page constants.
  CONSTANT_1: true,
  CONSTANT_2: 10,

  // Page methods.
  method1: function() {

  },
  method2: function() {

  },

  // Initialization method.
  init: function() {

  }
}

// Invoke the initialization method after the page loads.
addLoadEvent(Namespace.PageName.init);

    4.2.2 有私有成员的单体

    4.2.2.1 用下划线表示私有

    将所有的私有属性和私有方法都用下划线_开头，以此来告诫其他程序员这是内部使用的。

    GiantCorp.DataParser = {
  // Private methods.
  _stripWhitespace: function(str) {
    return str.replace(/\s+/, '');
  },
  _stringSplit: function(str, delimiter) {
    return str.split(delimiter);
  },
  
  // Public method.
  stringToArray: function(str, delimiter, stripWS) {
    if(stripWS) {
      str = this._stripWhitespace(str); //this调用内部方法
    }
    var outputArray = this._stringSplit(str, delimiter);//this调用内部方法
    return outputArray;
  }
};
    4.2.2.2 使用闭包实现私有

    在第3节使用闭包实现封装时，我们采用在构造函数里声明所有的私有变量和方法，然后采用this将特权方法提供给外部使用。这种方式，每次创建一个实例，所有的方法和属性都会各自保存一份，十分浪费内存。

    但是由于单体模式只有一个实例，每个方法和属性都只会创建一次，所以还是可以把所有的私有成员写在构造函数里。这种模式又称为模块模式

    //用立即执行的函数来创建单体
//这个函数创建了一个用来添加私有成员的闭包，在立即函数执行结束后，由于单体对象和私有成员同处于一个作用域，所有可以通过单体对象访问私有成员

GiantCorp.DataParser = (function() {
  // Private attributes.
  var whitespaceRegex = /\s+/;
  
  // Private methods.
  function stripWhitespace(str) {
    return str.replace(whitespaceRegex, '');
  }
  function stringSplit(str, delimiter) {
    return str.split(delimiter);
  }
  
  // Everything returned in the object literal is public, but can access the
  // members in the closure created above.
  return { 
    // Public method.
    stringToArray: function(str, delimiter, stripWS) {
      if(stripWS) {
        str = stripWhitespace(str);//访问私有成员
      }
      var outputArray = stringSplit(str, delimiter);
      return outputArray;//访问私有成员
    }
  };
})(); // Invoke the function and assign the returned object literal to 
     // GiantCorp.DataParser.

    4.3 惰性实例化

    上面4.2.2.2中使用闭包实现私有时是借助立即执行函数来完成的，这种方式会在脚本加载时就立即实例化。这对于那些被用作命名控件、特定网页专用代码包装器、组织相关使用方法的单体可以使用，但是对于某些加载了大量数据的单体，我们可以将实例化推迟到使用它的时候。而这种技术叫作惰性加载。

    具体实现方法就是，将之前的立即执行函数返回的单体对象修改为一个包含控制单体实例化的普通对象{getInstance}，然后再在需要时手动调用普通对象的方法完成单体对象的实例化

    MyNamespace.Singleton = (function() {
  
  var uniqueInstance; // Private attribute that holds the single instance.
  
  function constructor() { // All of the normal singleton code goes here.
    ...
  }
  
  return {
    getInstance: function() {
      if(!uniqueInstance) { // Instantiate only if the instance doesn't exist.
        uniqueInstance = constructor();
      }
      return uniqueInstance;
    }
  }
})();
    
//在需要时调用，进行单体对象的实例化
 MyNamespace.Singleton.getInstance() 
//访问单体对象的方法
MyNamespace.Singleton.getInstance() .method()

    4.4 分支

    分支：把浏览器之间的差异封装到能够在运行期间设置的动态方法中。这种技术可以减少浏览器嗅探和对象检测的次数，加快调用效率。

    分支**适用于任何需要在运行时才能确定的具体实现的情况**。

    4.4.1 通过单体对象实现

    前面在用闭包创建私有成员时，就是在运行时创建的。我们可以针对不同的浏览器设置不同的单体对象，然后在立即执行函数里根据运行时的状态返回特定的对象。

    由于需要针对不同的情况提供不同的单体对象，这将会占用更多的内存。需要在内存和计算时间上做一定的平衡。

    /* Branching Singleton (skeleton). */

MyNamespace.Singleton = (function() {
  var objectA = {
    method1: function() {
      ...
    },
    method2: function() {
      ...
    }
  };
  var objectB = {
    method1: function() {
      ...
    },
    method2: function() {
      ...
    }
  };
    //两个单体对象的接口是一致的，所以对于外部调用没有区别，不同的是内部的实现方法
  return (someCondition) ? objectA : objectB;
})();

    4.4.2 使用惰性函数实现

    当我们每次都需要进行条件判断，其实只需要判断一次，接下来的使用方式都不会发生改变的时候，想想是否可以考虑使用惰性函数。这种技术又叫作memorize

    //在判断一次后，就立即修改函数，后续就不需要再次试探
function addEvent (type, el, fn) {
    if (window.addEventListener) {
        addEvent = function (type, el, fn) {
            el.addEventListener(type, fn, false);
        }
    }
    else if(window.attachEvent){
        addEvent = function (type, el, fn) {
            el.attachEvent('on' + type, fn);
        }
    }
    
    addEvent(type, el, fn) //调用一次，给目标元素绑定好事件
}

    4.4.3 用工厂模式实现

    用工厂函数根据不同的浏览器情况创建不同的实例对象

    /* AjaxHandler interface. */

var AjaxHandler = new Interface('AjaxHandler', ['request', 'createXhrObject']);

/* SimpleHandler class. */

var SimpleHandler = function() {}; // implements AjaxHandler
SimpleHandler.prototype = {
  request: function(method, url, callback, postVars) {//请求的逻辑过程是固定的
    var xhr = this.createXhrObject();//不同的是创建实例对象的方法
    xhr.onreadystatechange = function() {
      if(xhr.readyState !== 4) return;
      (xhr.status === 200) ? 
        callback.success(xhr.responseText, xhr.responseXML) : 
        callback.failure(xhr.status);
    };
    xhr.open(method, url, true);
    if(method !== 'POST') postVars = null;
    xhr.send(postVars);
  },
  createXhrObject: function() { // Factory method.工厂方法
    var methods = [
      function() { return new XMLHttpRequest(); },
      function() { return new ActiveXObject('Msxml2.XMLHTTP'); },
      function() { return new ActiveXObject('Microsoft.XMLHTTP'); }
    ];
    
    for(var i = 0, len = methods.length; i < len; i++) {//首次会遍历所有可用的方法
      try {
        methods[i]();
      }
      catch(e) {
        continue;
      }
      // If we reach this point, method[i] worked.
      this.createXhrObject = methods[i]; // Memoize the method.一旦检测到方法可用就直接修改掉，后续不需要再检测
      return methods[i];
    }
    
    // If we reach this point, none of the methods worked.
    throw new Error('SimpleHandler: Could not create an XHR object.');
  } 
};

/* Usage. */

var myHandler = new SimpleHandler();
var callback = { 
  success: function(responseText) { alert('Success: ' + responseText); }, 
  failure: function(statusCode) { alert('Failure: ' + statusCode); } 
};
myHandler.request('GET', 'script.php', callback);

    4.4.4 示例

    根据不同的浏览器创建不同的XHR对象

    /* SimpleXhrFactory singleton, step 2. */

var SimpleXhrFactory = (function() {
  
  // The three branches.
  var standard = {
    createXhrObject: function() {
      return new XMLHttpRequest();
    }
  };
  var activeXNew = {
    createXhrObject: function() {
      return new ActiveXObject('Msxml2.XMLHTTP');
    }
  };
  var activeXOld = {
    createXhrObject: function() {
      return new ActiveXObject('Microsoft.XMLHTTP');
    }
  };
  
  //根据不同的情况返回不同的分支
  // To assign the branch, try each method; return whatever doesn't fail.
  var testObject;
  try {
    testObject = standard.createXhrObject();
    return standard; // Return this if no error was thrown.
  }
  catch(e) {
    try {
      testObject = activeXNew.createXhrObject();
      return activeXNew; // Return this if no error was thrown.
    }
    catch(e) {
      try {
        testObject = activeXOld.createXhrObject();
        return activeXOld; // Return this if no error was thrown.
      }
      catch(e) {
        throw new Error('No XHR object found in this environment.');
      }
    }
  }

})();

    5.方法的链式调用

    链式调用的本质是每个方法调用后通过return this返回同一个对象的引用，然后利用对象的点操作符.不断的继续调用方法。其中链式调用的开头就是创建出一个实例对象，后续的方法都是这个对象来调用的。

    5.1 链式调用构造方法

    (function(){
    
 function _$ (els)  //构造函数
    this.elements = []
    for (var i = 0, len = els.length; i < len; ++i) {
      var element = els[i]
      if (typeof element == 'string') {
        element = document.getElementById(element)
      }
      this.elements.push(element)
    }
  }
  _$.prototype = {//实例对象可以继承的方法
    each: function (fn) {
      for (var i = 0, len = this.elements.length; i < len; ++i) {
        fn.call(this, this.elements[i])
      }
      return this //返回实例对象的引用
    },
    setStyle: function (prop, val) {
      this.each(function (el) {
        el.style[prop] = val
      })
      return this //返回实例对象的引用，其他数据怎么存
    },
    show: function () {
      var that = this
      this.each(function (el) {
        that.setStyle('display', 'block')
      })
      return this //返回实例对象的引用
    },
    addEvent: function (type, fn) {
      var add = function (el) {
        if (window.addEventListener) {
          el.addEventListener(type, fn, false)
        }
        else if (window.attachEvent) {
          el.attachEvent('on' + type, fn)
        }
      }
      this.each(function (el) {
        add(el)
      })
      return this //
    }
  }
        
    window.$=function(){
      return  new _$(arguments) //用于给外部使用，链式调用的开头
    }


})()

//调用
$(window).addEvent('load', function () {
  $('test-1', 'test-2').show().
    setStyle('color', 'red').//方法调用时是直接传参，但是如果想将上一步的数据传给下一个函数怎么实现？
    addEvent('click', function (e) {
      $(this).setStyle('color', 'green')
    })

     //方法一：固定$方法在window下，方法名叫$，库外要使用该方法，只能使用$，而且容易出现冲突
    window.$=function(){
      return  new _$(arguments) //用于给外部使用，链式调用的开头
    }


//方法二：由程序员在使用前自行设置方法所在的命名空间和方法名，具有灵活性
window.installHelper = function(scope, interface) {
    scope[interface] = function() {
      return new _$(arguments);
    }
  };

//使用
installHelper(window,'$')
$('example').show()

    5.2 利用回调函数从链式调用的方法返回数据

    上一步中，所有实例对象的方法都通过return this返回了引用，但是如果想将某一个方法的执行结果传给下一个函数，如何实现？

    在取值器和赋值器中添加return this实现链式调用

    window.API2 = window.API2 || {};
API2.prototype = function() {
  var name = 'Hello world';
  // Privileged mutator method.
  setName: function(newName) {
    name = newName;
    return this;//返回实例对象的引用
  },
  // Privileged accessor method.
  getName: function(callback) {
    callback.call(this, name);
    return this;//返回实例对象的引用
  }
}();

// Implementation code.
var o2 = new API2;
o2.getName(console.log).setName('Meow').getName(console.log);

    6.工厂模式

    完成某个任务的整体逻辑流程是不变的，但是任务的某一个环节需要根据具体的运行状态来创建不同的实例对象（这些对象会实现同一个接口，不同的是接口中方法的实现细节）。这种情况就比较适用于工厂模式。

    适用场合：

    1. 动态实现
    2. 节省设置开销，将具体的实际设置代码集中在同一个地方

    6.1 简单的工厂模式

    将根据具体的运行状态来创建不同的实例对象的代码保存在一个对象中

    /* BicycleFactory namespace. */
//工厂对象，将根据具体的运行状态来创建不同的实例对象
var BicycleFactory = {
  createBicycle: function(model) {
    var bicycle;
    
    switch(model) {
      case 'The Speedster':
        bicycle = new Speedster();
        break;
      case 'The Lowrider':
        bicycle = new Lowrider();
        break;
      case 'The Comfort Cruiser':
      default:
        bicycle = new ComfortCruiser();
    }
    
    Interface.ensureImplements(bicycle, Bicycle);//实现同一接口
    return bicycle;
  }
};

/* BicycleShop class, improved. */
//整个卖自行车的逻辑是一致的，都需要经过assemble，wash，返回
var BicycleShop = function() {};
BicycleShop.prototype = {
  sellBicycle: function(model) {
    var bicycle = BicycleFactory.createBicycle(model);
    
    bicycle.assemble();
    bicycle.wash();
    
    return bicycle;
  }
};

var SimpleHandler = function() {}; // implements AjaxHandler

  }

var OfflineHandler = function() { // implements AjaxHandler
  this.storedRequests = [];

extend(OfflineHandler, SimpleHandler);

var XhrManager = {
  createXhrHandler: function() {
    var xhr;
    if(this.isOffline()) {

    }

  },

    ...                   // it succeeds.

    7.桥接模式

    作用：将抽象与实现隔离开来，以便两边可以独立变化。这对于JavaScript的事件驱动很有帮助。

    一个良好的API应该是针对接口而不是实现进行编程，不依赖于某种特定的实现环境

    7.1 事件监听器

    //使用桥接模式之前
addEvent(element, 'click', getBeerById);
function getBeerById(e) {
  var id = this.id;//与事件对象明确的绑定在了一起，
  asyncRequest('GET', 'beer.uri?id=' + id, function(resp) {
    // Callback response.
    console.log('Requested Beer: ' + resp.responseText);
  });
}

    //使用桥接模式之后
function getBeerById(id, callback) {//将id和callback作为实参，API更具有灵活性，在测试时也更加容易，不需要触发特定的事件即可测试
  // Make request for beer by ID, then return the beer data.
  asyncRequest('GET', 'beer.uri?id=' + id, function(resp) {
    // callback response
    callback(resp.responseText);
  });
}

addEvent(element, 'click', getBeerByIdBridge);
function getBeerByIdBridge (e) {
  getBeerById(this.id, function(beer) {
    console.log('Requested Beer: '+beer);
  });
}

    8.组合模式

    专为**创建web上的动态用户界面**而量身定做的模式

    组合模式擅长对大批对象进行操作，它专为组织这类对象并将操作从一层次传递到下一层次而设计

    该模式的优点：

    1. 对组合对象执行的操作将会向下传递到其子对象，然后子对象也会执行同样的操作；这有助于弱化各个对象的耦合
    2. 一批对象将形成树形结构，每个组合对象都有一个获取子对象的方法；这样就可以隐藏实现细节；

    8.1 组合对象的结构

    组合对象和构成它的子对象将形成树形结构。这个结构中共有两种对象：组合对象和叶对象。叶对象是组合对象最基本的元素，也是最终各个操作的落实点。

    组合对象和叶对象只是所在层次不同，并没有任何继承关系，他们应该实现相同的接口。

    8.2 表单验证

    需求：

    1. 希望能够通过对CompositeForm执行Save操作，能自动对该表单中的所有域也执行Save操作；
    2. 有两个FieldSet可以单独控制自己的子对象执行同样的操作，不影响其他的叶对象

    通过分析，可知整个组合模式的组织结构如下图所示：

    由于每个对象都应该实现同样的接口，这里我们定义了两个接口：Composite接口方便组合对象添加、删除、获取子对象；FormItem接口方便对所有对象执行同样的操作。总共需要实现的类如下图所示。

    //接口
var Composite = new Interface('Composite', ['add', 'remove', 'getChild']);
var FormItem = new Interface('FormItem', ['save']);

    /* CompositeForm class. */

var CompositeForm = function(id, method, action) { // implements Composite, FormItem
  this.formComponents = [];//保存该组合对象的子对象

  this.element = document.createElement('form');
  this.element.id = id;
  this.element.method = method || 'POST';
  this.element.action = action || '#';
};

CompositeForm.prototype.add = function(child) {
  Interface.ensureImplements(child, Composite, FormItem);//确保子对象都实现了同样的接口
  this.formComponents.push(child);
  this.element.appendChild(child.getElement());//添加到DOM元素中
};

CompositeForm.prototype.remove = function(child) {
  for(var i = 0, len = this.formComponents.length; i < len; i++) {
    if(this.formComponents[i] === child) {
      this.formComponents.splice(i, 1); // Remove one element from the array at 
                                        // position i.
      break;
    }
  }
};

CompositeForm.prototype.getChild = function(i) {
  return this.formComponents[i];
};

CompositeForm.prototype.save = function() {
  for(var i = 0, len = this.formComponents.length; i < len; i++) {
    this.formComponents[i].save();//遍历该组合对象的所有子对象，执行save操作
  }
};

CompositeForm.prototype.getElement = function() { 
  return this.element; 
};

    /* CompositeFieldset class. */
//中间层组合对象

var CompositeFieldset = function(id, legendText) { // implements Composite, FormItem
  this.components = {};//用对象存层次结构

  this.element = document.createElement('fieldset');
  this.element.id = id;

  if(legendText) { // Create a legend if the optional second 
                   // argument is set.
    this.legend = document.createElement('legend');
    this.legend.appendChild(document.createTextNode(legendText);
    this.element.appendChild(this.legend);
  }
};

//实现Composite
CompositeFieldset.prototype.add = function(child) {
  Interface.ensureImplements(child, Composite, FormItem);
  this.components[child.getElement().id] = child;
  this.element.appendChild(child.getElement());
};

CompositeFieldset.prototype.remove = function(child) {
  delete this.components[child.getElement().id];
};

CompositeFieldset.prototype.getChild = function(id) {
  if(this.components[id] != undefined) {
    return this.components[id];
  }
  else {
    return null;
  }
};

//实现FormItem
CompositeFieldset.prototype.save = function() {
  for(var id in this.components) {
    if(!this.components.hasOwnProperty(id)) continue;
    this.components[id].save();
  }
};

CompositeFieldset.prototype.getElement = function() { 
  return this.element; 
};

    /* Field class, abstract. */
//所有叶对象的父类，方便将各个叶对象相同的方法集中在一起，不同的再单独重写
//也可以不使用该类
var Field = function(id) { // implements Composite, FormItem
  this.id = id;
  this.element;
};

//实现Composite
Field.prototype.add = function() {};
Field.prototype.remove = function() {};
Field.prototype.getChild = function() {};

//实现FormItem
Field.prototype.save = function() {
  setCookie(this.id, this.getValue);
};

Field.prototype.getElement = function() { 
  return this.element; 
};

//由叶对象继承后重写
Field.prototype.getValue = function() { 
  throw new Error('Unsupported operation on the class Field.'); 
};

    //叶对象
/* InputField class. */

var InputField = function(id, label) { // implements Composite, FormItem
  Field.call(this, id);//类式继承 Field类

  this.input = document.createElement('input');
  this.input.id = id;

  this.label = document.createElement('label');
  var labelTextNode = document.createTextNode(label);
  this.label.appendChild(labelTextNode);

  this.element = document.createElement('div');
  this.element.className = 'input-field';
  this.element.appendChild(this.label);
  this.element.appendChild(this.input);
};
extend(InputField, Field); // Inherit from Field.

InputField.prototype.getValue = function() { //重写getValue
  return this.input.value;
};

    /* TextareaField class. */

var TextareaField = function(id, label) { // implements Composite, FormItem
  Field.call(this, id);//类式继承 Field类

  this.textarea = document.createElement('textarea');
  this.textarea.id = id;

  this.label = document.createElement('label');
  var labelTextNode = document.createTextNode(label);
  this.label.appendChild(labelTextNode);

  this.element = document.createElement('div');
  this.element.className = 'input-field';
  this.element.appendChild(this.label);
  this.element.appendChild(this.textarea);
};
extend(TextareaField, Field); // Inherit from Field.

TextareaField.prototype.getValue = function() { //重写getValue
  return this.textarea.value;
};
